# AWS Infrastructure Requirements for Buildora Custom Domains

The Buildora application provisions CloudFront distributions and ACM (SSL) certificates dynamically for user custom domains. To support this, the AWS team must provide an IAM User/Role and a dedicated S3 bucket configured as follows.

---

## 1. Create the S3 Bucket

1. Create a new S3 bucket (e.g., `buildora-published-sites-prod`).
2. This bucket will store static website files generated by the application.
3. **Access Configuration**: CloudFront needs to be able to read files from this bucket. Please attach a Bucket Policy that allows `s3:GetObject` either publicly, or restrict it via Origin Access Control (OAC) for `Service: cloudfront.amazonaws.com`.

---

## 2. Configure IAM Access

The backend application requires programmatic access to orchestrate AWS services.
Create an IAM User (with Access Keys) or an EC2/ECS IAM Role, and attach the following inline JSON policy:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "ACMCertificateManagement",
            "Effect": "Allow",
            "Action": [
                "acm:RequestCertificate",
                "acm:DescribeCertificate",
                "acm:DeleteCertificate"
            ],
            "Resource": "*",
            "Condition": {
                "StringEquals": {
                    "aws:RequestedRegion": "us-east-1"
                }
            }
        },
        {
            "Sid": "CloudFrontManagement",
            "Effect": "Allow",
            "Action": [
                "cloudfront:CreateDistribution",
                "cloudfront:GetDistribution",
                "cloudfront:GetDistributionConfig",
                "cloudfront:UpdateDistribution",
                "cloudfront:DeleteDistribution",
                "cloudfront:CreateInvalidation"
            ],
            "Resource": "*"
        },
        {
            "Sid": "S3WebsitePublishing",
            "Effect": "Allow",
            "Action": [
                "s3:PutObject",
                "s3:GetObject",
                "s3:DeleteObject",
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::<YOUR_S3_SITES_BUCKET>",
                "arn:aws:s3:::<YOUR_S3_SITES_BUCKET>/*"
            ]
        }
    ]
}
```

> **CRITICAL**: The ACM (Certificate Manager) permissions MUST be restricted to the `us-east-1` (N. Virginia) region. CloudFront requires all SSL certificates to be created in `us-east-1` regardless of where the application or S3 bucket is hosted.

---

## 3. Network & DNS Setup

To support free platform subdomains (e.g., `school.buildora.app`):
- Create a wildcard **A Record** (`*.buildora.app`) in your DNS provider/Route53 that points to the IP Address of the VPS/Server where the Node.js application is running. Traffic for subdomains is routed locally via Nginx.

---

## 4. Deliverables for the Backend Team

Once the infrastructure is ready, please securely provide the following details to the development team to inject into their `.env` file:

1. **`AWS_ACCESS_KEY_ID`** & **`AWS_SECRET_ACCESS_KEY`** (If using an IAM user instead of an instance role).
2. **`S3_REGION`** (e.g., `us-east-1`, `eu-central-1`).
3. **`S3_SITES_BUCKET`** (The exact name of the created S3 bucket).
4. **`CLOUDFRONT_PRIMARY_DISTRIBUTION_ID`** (If a baseline CloudFront distribution was created for the primary app domain, provide its ID. If not, provide the word `PENDING` to bypass application startup checks).
